# Tag Factory v1.2 版本开发计划

## 版本目标
完成标签基础管理功能，包括标签定义、层级结构管理、分类分组和树形结构展示。

---

## 一、数据库模型设计

### 1.1 标签表 (tagfactory_tags)
```
字段：
- id: 主键，自增
- name: 标签名称 (STRING(100), NOT NULL)
- code: 标签编码 (STRING(50), UNIQUE, NOT NULL)
- parent_id: 父标签ID (INTEGER, 可为空，支持层级)
- type: 标签类型 (ENUM: '分类', '业务', '自定义')
- color: 标签颜色 (STRING(20), 默认#1890ff)
- description: 标签描述 (TEXT)
- status: 状态 (TINYINT, 默认1: 启用)
- sort_order: 排序顺序 (INTEGER, 默认0)
- created_at: 创建时间
- updated_at: 更新时间
```

### 1.2 标签分组表 (tagfactory_tag_groups)
```
字段：
- id: 主键，自增
- name: 分组名称 (STRING(100), NOT NULL)
- code: 分组编码 (STRING(50), UNIQUE, NOT NULL)
- description: 分组描述 (TEXT)
- color: 分组颜色 (STRING(20))
- parent_id: 父分组ID (INTEGER，支持层级)
- sort_order: 排序顺序 (INTEGER, 默认0)
- status: 状态 (TINYINT, 默认1)
- created_at: 创建时间
- updated_at: 更新时间
```

### 1.3 标签与分组关联表 (tagfactory_tag_group_relations)
```
字段：
- id: 主键，自增
- tag_id: 标签ID (INTEGER, NOT NULL)
- group_id: 分组ID (INTEGER, NOT NULL)
- created_at: 创建时间
```

## 二、API接口设计

### 2.1 标签管理API (/api/tags)

**GET** - 获取标签列表
- 支持分页、搜索、筛选
- 返回树形结构数据
- 参数: page, pageSize, name, type, status, parent_id

**POST** - 创建标签
- 请求体: name, code, parent_id, type, color, description, sort_order

### 2.2 标签详情API (/api/tags/[id])

**GET** - 获取标签详情

**PUT** - 更新标签
- 请求体: name, code, parent_id, type, color, description, sort_order, status

**DELETE** - 删除标签
- 支持级联删除子标签

### 2.3 标签分组API (/api/tag-groups)

**GET** - 获取分组列表
- 支持树形结构返回

**POST** - 创建分组

### 2.4 标签操作API (/api/tags/[id]/move)
- 移动标签到其他父节点

### 2.5 标签批量操作API (/api/tags/batch)
- 批量移动、删除、更新状态
