# Tag Factory 1.7 版本开发计划

## 版本目标
实现完整的后台用户管理功能，支持多管理员操作和细粒度权限控制

## 功能范围

### 1. 用户管理
- 用户列表查询（支持分页、搜索、筛选）
- 用户创建接口
- 用户编辑接口（基本信息、密码、角色、状态）
- 用户删除/禁用接口
- 用户详情接口
- 批量操作接口（批量启用、禁用、删除）
- 登录日志查看

### 2. 角色权限管理（完善现有API）
- 角色CRUD接口完善
- 权限配置接口完善
- 用户角色分配
- 角色管理前端页面完善

### 3. 操作日志
- 操作日志记录（已存在模型，需完善记录逻辑）
- 日志查询接口
- 日志导出功能

### 4. 前端界面
- 用户管理页面（列表、创建/编辑弹窗、详情抽屉）
- 角色管理页面完善
- 权限配置界面
- 登录日志查看页面
- 操作日志查看页面

---

## 开发步骤

### 第一步：数据库模型扩展
1. 检查并扩展用户模型（如需要添加额外字段）
2. 创建登录日志表模型（login_logs）
3. 完善操作日志记录逻辑

### 第二步：用户管理API
1. 创建 `/api/users/route.ts` - 用户列表查询、创建用户
2. 创建 `/api/users/[id]/route.ts` - 用户详情、编辑、删除
3. 创建 `/api/users/[id]/reset-password` - 密码重置
4. 创建 `/api/users/batch-action` - 批量操作
5. 创建 `/api/users/login-logs` - 登录日志查询

### 第三步：角色权限API完善
1. 完善 `/api/roles/[id]/route.ts` - 角色编辑
2. 完善 `/api/roles/[id]/permissions/route.ts` - 权限配置
3. 创建用户角色分配接口

### 第四步：操作日志API
1. 完善审计日志记录中间件
2. 完善 `/api/audit-logs/route.ts` - 日志查询
3. 完善 `/api/audit-logs/export/route.ts` - 日志导出

### 第五步：前端界面开发
1. 创建用户管理页面 `/src/app/users/page.tsx`
2. 创建用户表单组件 `/src/app/users/UserForm.tsx`
3. 创建登录日志页面 `/src/app/users/login-logs/page.tsx`
4. 完善角色管理页面
5. 添加操作日志查看入口
6. 更新导航菜单添加用户管理入口

### 第六步：测试与验证
1. API接口测试
2. 前端功能测试
3. 修复发现的问题
