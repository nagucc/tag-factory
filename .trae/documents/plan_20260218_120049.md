## 三、技术实现方案

### 3.1 数据库设计

**AI打标任务表 (tf_ai_tag_tasks)**
| 字段 | 类型 | 说明 |
|------|------|------|
| id | BIGINT | 主键 |
| work_plan_id | BIGINT | 工作计划ID |
| prompt | TEXT | 打标提示词 |
| page_size | INT | 每页记录数 |
| total_records | INT | 总记录数 |
| success_count | INT | 成功数 |
| fail_count | INT | 失败数 |
| status | ENUM | pending/running/completed/failed |
| result_json | JSON | 打标结果详情 |
| user_id | BIGINT | 操作人ID |
| created_at | DATETIME | 创建时间 |

**提示词模板表 (tf_prompt_templates)**
| 字段 | 类型 | 说明 |
|------|------|------|
| id | BIGINT | 主键 |
| name | VARCHAR(200) | 模板名称 |
| category | VARCHAR(50) | 分类 |
| prompt | TEXT | 提示词内容 |
| description | TEXT | 描述 |
| is_preset | BOOLEAN | 是否预设 |
| use_count | INT | 使用次数 |
| created_by | BIGINT | 创建人 |
| created_at | DATETIME | 创建时间 |

### 3.2 环境变量配置
```
OPENAI_API_KEY=sk-xxx
OPENAI_MODEL=gpt-4o
OPENAI_TEMPERATURE=0.7
OPENAI_MAX_TOKENS=2000
```

### 3.3 AI打标流程

**预览流程**：
1. 用户输入提示词、选择记录范围、设置每页记录数
2. 后端将记录分批（每批page_size条）
3. 每批构建提示词调用OpenAI API
4. 收集所有批次推荐结果返回前端
5. 不写入数据库

**执行流程**：
1. 用户确认预览结果后点击执行
2. 后端遍历预览结果
3. 调用批量打标接口写入数据库
4. 更新工作计划统计
