## 二、详细开发计划

### 阶段1：数据库模型扩展（第1天）

**1.1 创建审计日志模型**
```
表: tf_audit_logs
- id: BIGINT 主键
- user_id: BIGINT 用户ID
- action: STRING 操作类型
- resource_type: STRING 资源类型
- resource_id: BIGINT 资源ID
- details: JSON 操作详情
- ip_address: STRING IP地址
- user_agent: STRING 用户代理
- created_at: DATETIME
```

**1.2 扩展权限模型**
```
表: tf_permissions (扩展)
- resource_type: STRING 资源类型
- action: STRING 操作类型
- description: STRING 描述
```

**1.3 创建系统指标模型**
```
表: tf_system_metrics
- id: BIGINT
- metric_type: STRING (cpu/memory/disk/network)
- metric_value: FLOAT
- recorded_at: DATETIME
```

### 阶段2：审计日志功能（第2-3天）

**2.1 审计日志API**
- `GET /api/audit-logs` - 查询审计日志列表
- 支持分页、按用户/时间/操作类型筛选

**2.2 审计日志中间件**
- 创建审计日志记录中间件
- 自动记录关键操作（登录、创建、更新、删除）

**2.3 审计日志页面**
- `/audit-logs/page.tsx` - 审计日志列表页
- 使用Ant Design Table展示
- 支持筛选和导出

### 阶段3：权限管理系统（第4-6天）

**3.1 权限模型扩展**
- 实现资源+操作的细粒度权限
- 权限定义格式：`resource:action` (如 `data-sources:read`)

**3.2 权限API**
- `GET /api/permissions` - 获取所有权限
- `GET /api/roles/[id]/permissions` - 获取角色权限
- `PUT /api/roles/[id]/permissions` - 更新角色权限

**3.3 角色管理页面**
- `/roles/page.tsx` - 角色列表
- 角色权限配置（使用Tree组件）

**3.4 权限检查中间件**
- 在API路由中检查权限

### 阶段4：第三方认证集成（第7-8天）

**4.1 OAuth2.0认证**
- 创建OAuth适配器
- 支持GitHub/Google登录
- 集成到登录流程

**4.2 认证配置页面**
- 在系统配置中添加OAuth配置
- 启用/禁用第三方登录

### 阶段5：系统监控模块（第9-10天）

**5.1 指标采集服务**
- CPU、内存、磁盘、网络指标采集
- 定时任务收集系统指标

**5.2 监控API**
- `GET /api/system/metrics` - 获取当前指标

**5.3 监控仪表盘**
- `/monitoring/page.tsx` - 系统监控页面
- 使用Ant Design Charts展示图表

### 阶段6：告警机制（第11天）

**6.1 告警规则配置**
- 告警阈值设置
- 告警通知方式配置

**6.2 告警服务**
- 定时检查指标超阈值
- 发送告警通知

### 阶段7：性能优化与安全加固（第12天）

**7.1 性能优化**
- 添加常用查询缓存
- 优化数据库索引
- 静态资源压缩

**7.2 安全加固**
- 输入验证增强
- SQL注入防护
- XSS防护

### 阶段8：测试与文档（第13-14天）

**8.1 测试**
- 单元测试覆盖
- 集成测试
- API接口测试

**8.2 文档**
- 更新API文档
- 编写部署文档

## 五、技术实现要点

1. **审计日志**：使用Sequelize模型+中间件自动记录
2. **权限系统**：扩展现有Permission模型，资源+操作二维权限
3. **OAuth2.0**：使用passport.js或自定义实现
4. **系统监控**：使用os模块和node-os-utils采集指标
5. **前端UI**：使用Ant Design Charts展示图表
